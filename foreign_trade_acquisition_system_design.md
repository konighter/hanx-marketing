# 外贸获客系统规划与架构设计文档

## 1. 系统概述
本系统旨在打造一个全自动化的“外贸获客雷达”，通过整合**海关数据**的精准供应链历史、**Google Maps** 的本地化商家名录，并结合前沿的 **GEO（生成式引擎优化）** 模块，实现从“找客户”、“挖联系人”到“AI 占位”的全链路闭环获客系统。

---

## 2. 系统功能规划

### 第一阶段：海关数据分析模块（核心获客雷达）
**目标：** 通过真实的进出口记录锁定“大客户”及“竞争对手的客户”。
*   **多维检索中心**
    *   关键词/HS编码搜索：支持模糊匹配产品名称或 6/8 位 HS 编码。
    *   交易时段筛选：追踪近 1-3 年的采购频次，识别“活跃买家”与“沉睡买家”。
*   **买家深度画像**
    *   采购偏好分析：自动统计买家最常采购的规格、单价区间及来源国。
    *   供应链关系网：可视化展示该买家所有的供应商（寻找“挖墙脚”切入点）。
*   **动态监测与预警**
    *   新买家发现：行业内首次出现进口记录的公司自动推送到仪表盘。
    *   流失预警：老客户超过 3 个月未采购或转向其他供应商时触发提醒。

### 第二阶段：Google Maps 局部获客模块（精准落地与触达）
**目标：** 获取特定地区的线下分销商、批发商、零售商及其核心联系方式。
*   **区域化精准挖掘**
    *   坐标+半径爬取：针对特定工业区、商贸中心进行地毯式搜索。
    *   行业标签过滤：自动提取评分、营业状态（过滤已结业的无效点）。
*   **联系方式“深度还原”**
    *   官网解析引擎：自动从 Maps 关联的官网中提取邮箱、LinkedIn、WhatsApp 和 Facebook 链接。
    *   电话自动清洗：识别固定电话与手机号，支持一键跳转 WhatsApp 发送。
*   **实景与口碑分析**
    *   评论词云图：分析目标企业在 Maps 上的差评点，作为开发信切入点。

### 第三阶段：GEO（生成式引擎优化）模块（未来增长极）
**目标：** 让品牌成为 Perplexity、ChatGPT 等 AI 模型在回答采购咨询时的“首选推荐”。
*   **AI 语料库优化建议**
    *   高质量 Q&A 库生成：基于海关数据痛点，自动生成符合自然语言查询（NLQ）的问答内容。
    *   结构化数据 (Schema) 增强：为企业官网生成专门针对 AI 爬取的 JSON-LD 标签。
*   **权威性背书监测**
    *   第三方提及量追踪：监测品牌在 Reddit、Quora 等 AI 训练语料高频采集区的活跃度。
*   **GEO 效果仪表盘**
    *   AI 模拟问答追踪：自动测试主流 AI 模型搜索“Top suppliers for [product]”，监控品牌占位情况。

## 3. 技术架构设计 (Architecture Design)

### 3.1 模块化分层架构
*   **数据采集层 (Data Sourcing)**
    *   分布式爬虫集群（Scrapy / Playwright / 指纹浏览器防风控）。
    *   API 网关集成（海关数据源 API、第三方联系人富化 API）。
*   **数据中台层 (Data Pipeline & Storage)**
    *   数据清洗 ETL：基于 Python 进行非结构化数据的标准化与去重。
    *   存储方案：
        *   `ClickHouse`：用于海量的海关交易明细聚合与分析 (OLAP)。
        *   `Elasticsearch`：用于企业黄页、Maps 数据的毫秒级全文检索。
        *   `Vector DB (如 Milvus)`：用于构建语义图谱和相似客户推荐。
*   **AI 计算层 (LLM Agent)**
    *   知识抽取：从杂乱的网页和评论中提取关键意图数据。
    *   文本生成：结合客户画像自动生成高回复率的破冰开发信。
*   **业务逻辑层 (Business Logic & UI)**
    *   基于 Vue3 (Frontend) + Java/Spring Boot (Backend) 提供可视化看板、CRM 管控与流转配置。

### 3.2 核心数据流向
`外部数据源 (Web/Maps/Customs)` ➡️ `爬虫抓取/API拉取` ➡️ `ETL数据清洗与去重` ➡️ `入库 (ES/ClickHouse)` ➡️ `知识图谱/AI 画像关联` ➡️ `前端业务展示 (客户雷达/开发信队列)`
