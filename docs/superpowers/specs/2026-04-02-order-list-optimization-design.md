# 技术设计规约：多平台订单列表优化 (折叠 Shopee 风格 + 批量操作)

## 1. 目标描述描述
为了提升 ERP 用户处理多平台订单的效率，本次优化将对订单列表页进行重构。主要目标是：
- 采用更直观的“订单块”展示方式（Shopee 风格）。
- 通过“默认折叠、手动展开”机制平衡信息密度与深度。
- 引入批量操作栏，支持多订单快速处理。
- 精简筛选区域，聚焦核心业务维度。

## 2. 方案概览

### 2.1 筛选区域优化
精简搜索表单，仅保留以下核心字段：
- **店铺**：级联选择器（平台 + 店铺）。
- **配送渠道**：下拉选择（FBA / FBM 等）。
- **SKU**：文本输入（匹配本地 SKU 或买家 SKU）。
- **订单日期**：日期范围选择器。
- **订单号**：文本输入（支持模糊匹配）。

### 2.2 列表结构 (Folding Shopee Style)
- **容器**：放弃传统的 `el-table` 树形模式，改为基于列表的“任务块”布局。
- **状态页签 (Tabs)**：在列表顶部增加状态页签（全部、待处理、待出库、运送中、已完成、已取消）。
- **订单行 (Row/Card)**：
    - **折叠状态 (Default)**：展示平台图标、单号、所属店铺、主要 SKU 摘要（如 `SKU-A... +2`）、订单总额、订单状态。
    - **展开状态**：点击行展开，显示该订单下的所有商品详情（图片、SKU、标题、单价、数量）、详细财务分解（收入 $、佣金 $、配送费 $、预估利润 $）以及备注信息。

### 2.3 批量操作 (Batch Actions)
- 在筛选栏下方、列表上方增加一个**悬浮批量工具栏**。
- 当勾选一个或多个订单复选框时，工具栏变为激活状态。
- **功能**：批量同步订单、批量打印面单、批量导出。

## 3. 技术实现要点

### 3.1 前端组件
- **Vue 组件**：`index.vue`
- **主要逻辑**：
    - 使用 `v-for` 渲染订单块。
    - 使用一个响应式数组（如 `expandedRowKeys`）维护当前展开的订单 ID。
    - 级联状态管理：点击全选复选框，更新 `selectedOrderIds`。

### 3.2 样式系统 (Vanilla CSS)
- 采用高密度布局（Compact Layout），减小折叠行的高度。
- 使用渐变和阴影区分不同的订单状态。
- 增加平台专属标签色（如 Amazon: #ff9900）。

### 3.3 数据交互
- 复用现有的 `OrderApi.queryCrossOrderPage` 接口，但前端按需组装展示模型。
- `CrossOrderResp` 已包含 `orderItemList`，前端直接遍历展示即可。

## 4. 验证计划

### 自动化测试
- 验证筛选器在输入单个字段或组合字段时能正确返回结果。
- 验证点击“展开/收起”按钮时，DOM 正确切换且不丢失状态。
- 验证批量勾选后，点击“批量同步”调用的 API 参数正确。

### 手动验证
- 检查在 1080p 屏幕下，折叠状态下是否能看到 15 个及以上的订单。
- 确认订单详情中的金额换算（分 -> 元）准确无误。
