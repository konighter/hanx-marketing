状态协作模型 (Status Collaboration)

如何区分外部异步等待 (SUBMITTED) 与 本地执行 (PROCESSING)

任务流转全图 (Full Flow)

PENDING
待处理
➔
SUBMITTED
外部等待 (Amazon)

➔
SUCCESS
完成
SUBMITTED: 异步等待态

场景： 调用外部 API (如 SubmitReport) 后，对方还在排队。

行为： 引擎 释放线程，定时再次调用 onPoll()。

目的： 避免线程阻塞，提高并发能力。

PROCESSING: 执行中状态

场景： 报表已生成，本地正在下载、解析并落库 (Doris)。

行为： 引擎 占用线程，调用 onExecute() 进行实际计算。

目的： 防止其他节点并发执行同一任务，标识任务正在消耗本机资源。

结合方式建议

任务场景 流转路径
纯同步任务 (如 Token 刷新) PENDING ➔ PROCESSING ➔ SUCCESS
异步报表任务 (如 Amazon) PENDING ➔ SUBMITTED ➔ PROCESSING ➔ SUCCESS
