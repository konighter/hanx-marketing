任务状态机与钩子设计 (State Machine & Hooks)

针对异步任务扩展状态定义，实现更细粒度的控制

扩展后的标准状态 (Standard Statuses)

Status: PENDING

待处理 (New)

任务刚入库或排期中。引擎将调用 onStart()。

Status: SUBMITTED

已提交 (Async)

已向外部平台申请，正在等待结果。引擎将调用 onPoll()。

Status: SUCCESS

完成 (Terminal)

任务成功结束。如果是子任务，触发父任务检查。

Status: FAILED

失败 (Terminal)

任务异常，记录错误原因，根据策略决定是否重试。

处理器钩子设计 (Handler Hooks)

AdsTaskHandler 接口
public interface AdsTaskHandler {
    /**
     * 当状态为 PENDING 时调用
     * 如果是同步任务，直接返回 SUCCESS
     * 如果是异步任务，提交后返回 SUBMITTED
     */
    AdsTaskResult onStart(AdsSyncTaskDO task);

    /**
     * 当状态为 SUBMITTED 时调用 (轮询)
     * 返回 SUCCESS 代表完成，返回 SUBMITTED 代表继续等待
     */
    AdsTaskResult onPoll(AdsSyncTaskDO task);

    // 处理器标识
    String getTaskType();
}
