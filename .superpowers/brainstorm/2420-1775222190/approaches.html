Three Approaches for Unified Implementation

Comparing complexity vs flexibility

Low Complexity

Schema-Direct (Relay)

Frontend renders forms directly from Amazon's JSON Schema. Backend acts as a pass-through with validation.

  • Zero maintenance for new product types
  • Fastest to implement
  • Hard to customize UI for better UX
  • Querying across types is limited
Balanced

Metadata-Driven Service

Store schemas locally. Use a "Schema Service" to provide UI hints (which fields are Title/Price) while keeping the rest dynamic.

  • Consistent UX across all types
  • Powerful searching (hybrid SQL+JSON)
  • Requires schema caching/sync logic
High Complexity

Domain-Mapped (Canonical)

Map Amazon attributes to an internal "Master Product Model". Complex transformation logic for each type.

  • Best for Multi-platform sync
  • Perfect control over UI/UX
  • Huge maintenance overhead
  • Hard to keep up with Amazon updates

Question: How important is Multi-Platform sync?

If you plan to sync the same product data across Amazon, Shopee, and TikTok, Approach C is better. If you are focused primarily on Amazon-specific management, Approach B is more efficient.
