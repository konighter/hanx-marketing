Common Core + Platform Private Architecture

Separating cross-platform logic from platform-specific extensions

Data Flow: Amazon Listing Lifecycle
Unified Core (Standard)
ProductModel
• SKU / ASIN / MarketID
• Status / Category
• Title / Main Image
• Price / Inventory
Used for: Copy, Link, Mapping
↔
Platform Private (Amazon)
Dynamic Attributes JSON
• heel_height
• fabric_type
• battery_info
• (Validates via Amazon Schema)
Used for: Custom UI, API Calls

Operational Strategy

A
Parser Extraction

Extract "Common Core" fields from the Amazon JSON blob into structured columns for global searching/linking.

B
Raw Attribute Storage

Store the full Amazon attributes JSON in a dedicated "Attribute Table" for type-specific CRUD logic.

C
Schema-Driven Lifecycle

A "Schema Service" ensures the Private JSON is always valid against the Amazon-provided JSON Schema.
